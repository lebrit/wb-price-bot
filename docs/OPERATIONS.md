# Эксплуатация

## Проверка состояния

```bash
sudo wb-price-bot status
sudo wb-price-bot doctor
sudo wb-price-bot logs
```

Healthcheck проверяет SQLite и свежесть heartbeat мониторинга. После первого запуска Chromium
образ может стартовать дольше обычного; установщик ждёт до пяти минут.

## Обновление

`sudo wb-price-bot update` получает последний релиз из публичного GitHub-репозитория. Перед
переключением создаётся backup. Новый release разворачивается в отдельный каталог, собирается
отдельный image, затем атомарно переключается `current`. Если healthcheck не проходит, ссылка
и контейнер возвращаются на предыдущий релиз, а rollback также проверяется healthcheck.

Обновление берёт последний GitHub Release, а не произвольное состояние `main`.

## Backup и восстановление

Откройте `sudo wb-price-bot` → «Резервные копии». Восстановление проверяет
`PRAGMA integrity_check`, делает предвосстановительную копию, останавливает bot, атомарно
заменяет базу и ждёт healthcheck. При неудаче старая база возвращается.

Операторские копии находятся в `/var/backups/wb-price-bot`. Telegram token и Fernet key в
них не входят. Без исходного Fernet key зашифрованную WB-сессию расшифровать нельзя.

## Смена региона

Параметр `WB_DESTINATION` влияет на публичную цену и логистику. Узнайте `dest` из `xinfo`
Wildberries для нужного адреса и измените его в меню. После смены источник/регион должен
считаться новой контрольной точкой. Бот включает `dest` в идентификатор источника, поэтому
первая цена нового региона автоматически становится контрольной и не создаёт ложного сигнала.

## Реакция на ошибки WB

- `429` — увеличьте интервал;
- `403/498` после fallback через `curl` — остановите проверки и дождитесь восстановления доступа;
- account session error — снова выполните ручной capture;
- неизвестная схема — обновите адаптер и contract fixtures, не подставляйте цену `0`.
