# Эксплуатация

## Проверка состояния

```bash
sudo wb-price-bot status
sudo wb-price-bot doctor
sudo wb-price-bot logs
```

Healthcheck bot проверяет SQLite и свежесть heartbeat мониторинга, auth — HTTP и SQLite,
Caddy — валидность конфигурации. Chromium не работает постоянно: auth запускает один
временный экземпляр только на время входа пользователя. Контейнер auth ограничен одним CPU,
700 МБ RAM, 1100 МБ RAM+swap и 256 процессами, чтобы не уронить малый VPS при сбое Chromium.

## Домен и HTTPS

До установки создайте A/AAAA-запись домена на сервер. Caddy занимает `80/tcp`, `443/tcp` и
`443/udp`, автоматически получает и обновляет сертификат. Настройки находятся в
`AUTH_DOMAIN` и `AUTH_PUBLIC_URL`; изменить и проверить их можно в `sudo wb-price-bot` →
«Web-авторизация WB». Установщик не меняет firewall и не останавливает чужой
reverse proxy.

Проверка:

```bash
curl -fsS https://auth.example.com/health
sudo wb-price-bot doctor
```

## Обновление

`sudo wb-price-bot update` получает последний релиз из публичного GitHub-репозитория. Перед
переключением создаётся backup. Новый release разворачивается в отдельный каталог, собирается
отдельный image, затем атомарно переключается `current`. Если healthcheck не проходит, ссылка
и контейнер возвращаются на предыдущий релиз, а rollback также проверяется healthcheck.

Обновление берёт последний GitHub Release, а не произвольное состояние `main`.

## Backup и восстановление

Откройте `sudo wb-price-bot` → «Резервные копии». Восстановление проверяет
`PRAGMA integrity_check`, делает предвосстановительную копию, останавливает bot и auth, атомарно
заменяет базу и ждёт healthcheck. При неудаче старая база возвращается.

Операторские копии находятся в `/var/backups/wb-price-bot`. Telegram token и Fernet key в
них не входят. Без исходного Fernet key зашифрованную WB-сессию расшифровать нельзя.

## Смена региона

Параметр `WB_DESTINATION` задаёт серверный регион по умолчанию. Пользователь может отправить
геопозицию через `/settings`: бот однократно получает региональный `dest` у Wildberries и не
сохраняет координаты. После смены источник/регион должен
считаться новой контрольной точкой. Бот включает `dest` в идентификатор источника, поэтому
первая цена нового региона автоматически становится контрольной и не создаёт ложного сигнала.

## Пользовательские настройки и перенос списка

- `/settings` — регион, тихие часы и дневная сводка в `APP_TIMEZONE`;
- `/import` — до `MAX_BULK_IMPORT` ссылок или артикулов текстом/TXT;
- `/export` — CSV или JSON без токенов, сессий и Telegram-координат;
- `/folders`, `/folder ИМЯ`, `/tag ТЕГ` — организация и фильтры.

## Пользователи

`TELEGRAM_ALLOWED_USERS` — администраторы. В режиме `approval` новый пользователь после
`/start` получает `pending`, а администраторы — кнопки решения. Просмотр и повторное изменение:

```text
/users pending
/users approved
/users blocked
```

`open` одобряет нового пользователя автоматически; `allowlist` полностью запрещает заявки.
Блокировка сразу исключает товары пользователя из мониторинга и отменяет доставку его outbox.

## Web-авторизация WB

Пользователю сначала нужен хотя бы один добавленный товар. Затем `/account` создаёт связанную
с Telegram ID одноразовую Mini App-ссылку на 600 секунд. Подпись `initData` проверяется до
запуска браузера. Телефон и SMS-код передаются по TLS в один временный Chromium, но не
записываются в SQLite или журналы. Экран браузера не транслируется и удалённого управления нет.

Одновременно выполняется один вход; остальные открытые формы показывают очередь. После входа
Chromium открывает карточку добавленного товара, сохраняет отфильтрованную зашифрованную
сессию и сразу завершается. Дальнейшие штатные проверки используют лёгкий авторизованный
HTTP-запрос. При CAPTCHA вход завершается без её распознавания или обхода — пользователю нужно
повторить попытку позже.

Перезапуск `auth` прерывает текущую попытку, но не удаляет уже сохранённые WB-сессии. Новая
команда `/account` отменяет прежнее незавершённое окно этого пользователя.

## MPSTATS fallback

Откройте `sudo wb-price-bot` → «Лицензированный источник MPSTATS», добавьте API token и
запустите проверку. Token хранится в отдельном secret-файле. Источник вызывается только при
ошибке публичного WB endpoint и только для обычной цены артикула. Он не подменяет цену
конкретного размера или авторизованного аккаунта. Ответ старше `MPSTATS_MAX_AGE_HOURS`
отклоняется. Для работы нужна подходящая подписка MPSTATS.

## Реакция на ошибки WB

- `429` — увеличьте интервал;
- `403/498` после fallback через `curl` — остановите проверки и дождитесь восстановления доступа;
- account session error — переподключите аккаунт через `/account`;
- форма входа не открывается — проверьте DNS, сертификат, `/health` и открытие из личного чата;
- вход остановлен CAPTCHA — повторите позже; бот не распознаёт и не обходит эту проверку;
- MPSTATS authorization/stale-data error — проверьте token, тариф и время обновления;
- неизвестная схема — обновите адаптер и contract fixtures, не подставляйте цену `0`.
