# Безопасность

## Секреты

- Telegram token и Fernet key находятся в `/etc/wb-price-bot/secrets`, режим `0640`.
- Контейнер работает как UID/GID `10001`, без capabilities, с read-only root filesystem.
- WB browser storage state шифруется до записи в SQLite.
- Телефон, SMS-код, CAPTCHA и пароль приложение не принимает и не сохраняет.
- Browser storage state импортируется прямо по SSH/stdin или из локального файла сервера;
  Telegram-бот такие файлы не принимает.
- HTTP debug-логи отключены; токены и cookies не должны попадать в журнал.

Browser storage state позволяет действовать от имени пользователя. Относитесь к нему как к
паролю: передавайте только своему серверу, удаляйте исходный JSON после импорта и
переподключайте аккаунт при подозрении на утечку.

## Доступ

Идентификатором пользователя служит числовой Telegram ID, а не username. Вход разрешён только
ID из `TELEGRAM_ALLOWED_USERS`. Callback-действия повторно проверяют владельца товара.
Экран аккаунта доступен только в личном чате, а серверный импорт дополнительно проверяет, что
Telegram ID присутствует в allowlist.

## Ссылки

Парсер принимает только точный allowlist доменов WB. Каждый redirect короткой ссылки
проверяется отдельно и ограничен пятью переходами. Это предотвращает SSRF через присланную
ссылку.

## Удаление

Runtime-only и полный purge разделены и требуют разные контрольные фразы. Скрипт сверяет
канонический путь перед рекурсивным удалением и не удаляет Docker, чужие контейнеры или
чужие образы.

Уязвимости не публикуйте вместе с действующими токенами, cookies или дампом базы.
