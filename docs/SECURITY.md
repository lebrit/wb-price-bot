# Безопасность

## Секреты

- Telegram token, Fernet key и опциональный MPSTATS token находятся в
  `/etc/wb-price-bot/secrets`, режим `0640`.
- Контейнеры bot/auth работают как UID/GID `10001`, без capabilities, с read-only root filesystem.
- WB browser storage state шифруется до записи в SQLite.
- Телефон и SMS-код вводятся в одноразовой Telegram Mini App-форме и передаются по TLS через
  self-hosted сервер только в запущенный на время входа Chromium. Они не сохраняются в SQLite,
  файлах или журналах. Администратор сервера входит в модель доверия.
- Browser storage state импортируется прямо по SSH/stdin или из локального файла сервера;
  Telegram-бот такие файлы не принимает.
- HTTP debug-логи отключены; токены и cookies не должны попадать в журнал.
- Caddy не ведёт access log одноразовых URL и WebSocket; системные TLS/error-сообщения остаются.
- Telegram-координаты используются только для запроса регионального `dest` и не записываются
  в SQLite; сохраняются только `dest` и нейтральная подпись источника.
- Экспорт CSV/JSON содержит настройки товаров, но не содержит токены, WB-сессии и координаты.

Browser storage state позволяет действовать от имени пользователя. Относитесь к нему как к
паролю: передавайте только своему серверу, удаляйте исходный JSON после импорта и
переподключайте аккаунт при подозрении на утечку.

## Доступ

Идентификатором пользователя служит числовой Telegram ID, а не username. Админские права
даются только текущим ID из `TELEGRAM_ALLOWED_USERS` и сразу отзываются после изменения
конфигурации и перезапуска bot. В режимах `approval`/`open` вход разрешён также пользователям
со статусом `approved`; в режиме `allowlist` — только настроенным ID.
Callback-действия повторно проверяют владельца товара. Экран аккаунта доступен только в
личном чате.

Одноразовая ссылка содержит криптографически случайный ID, связана с Telegram ID, действует
10 минут и погашается транзакционно. До запуска браузера сервер проверяет HMAC-подпись и срок
действия Telegram Mini App `initData`. Второй пользователь не может использовать чужое окно.
Одновременно разрешён один временный Chromium; остальные попытки ожидают очередь.

Mini App принимает только телефон и числовой одноразовый код. Значения не включаются в URL,
ответы, access log или сообщения об ошибках и больше не используются после заполнения формы WB.
Перед сохранением сервер оставляет только cookies доменов WB, разрешённые origins, точный card
API URL и небольшой allowlist заголовков, затем шифрует состояние Fernet. Кадры страницы,
клавиатура и мышь через WebSocket не передаются. CAPTCHA не распознаётся и не обходится:
попытка завершается без сохранения сессии.

## Ссылки

Парсер принимает только точный allowlist доменов WB. Каждый redirect короткой ссылки
проверяется отдельно и ограничен пятью переходами. Это предотвращает SSRF через присланную
ссылку.

## Удаление

Runtime-only и полный purge разделены и требуют разные контрольные фразы. Runtime-only
сохраняет SQLite, конфигурацию, секреты и volumes сертификатов Caddy; полный purge удаляет их.
Скрипт сверяет
канонический путь перед рекурсивным удалением и не удаляет Docker, чужие контейнеры или
чужие образы.

Уязвимости не публикуйте вместе с действующими токенами, cookies или дампом базы.
