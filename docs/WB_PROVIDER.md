# Провайдер Wildberries

Состояние проверено 11 июля 2026 года. Официального buyer API нет; схема может измениться.

## Публичный режим

Используется `https://card.wb.ru/cards/v4/detail` с `nm` через `;`, валютой, языком и
региональным `dest`. Цена хранится в копейках и считается как `product + logistics`.
При добавлении бот показывает доступные размеры и закрепляет выбранный `optionId` и текущего
продавца `supplierId`. Если вариант не выбран, используется минимальная доступная цена.
Если выбранный размер или продавец исчез, проверка завершается ошибкой, а чужая цена не
подставляется. Ноль, `null`, пропавший товар и ошибочный JSON никогда не считаются снижением.

Telegram-геопозиция передаётся в `user-geo-data.wildberries.ru/get-geo-info`; полученный
`dest` применяется к публичным карточкам конкретного пользователя. Координаты после запроса
не сохраняются.

По умолчанию одна пачка содержит не более 25 артикулов, интервал — 30 минут, jitter — две
минуты. Если встроенный HTTP-клиент получает `403/498` из-за сетевого профиля, тот же
публичный endpoint один раз запрашивается системным `curl`. Если и он получает защитный
ответ, проверка прекращается: challenge не решается и цена не подменяется.

## Аккаунтный режим (beta)

Основной путь — `/account` в Telegram. После добавления хотя бы одного товара бот создаёт
одноразовую Mini App-ссылку, связанную с Telegram ID. Форма по TLS передаёт телефон и SMS-код
в один кратковременный Chromium на self-hosted сервере; эти значения не сохраняются и не
попадают в журналы. Устанавливать расширение или программу на компьютер не нужно.

После входа Chromium открывает карточку добавленного товара и фиксирует cookies, localStorage
и заголовки авторизованного card-запроса. Сервер проверяет строгий allowlist, шифрует storage
state и атомарно сохраняет его только для владельца одноразовой ссылки. Экран не транслируется,
удалённого управления нет, а при CAPTCHA попытка прекращается без распознавания или обхода.
`scripts/capture_wb_session.py` остаётся аварийным административным fallback.

На сервере адаптер сначала выполняет лёгкий card-запрос с сохранёнными Bearer/cookies и
использует ответ для цены выбранного размера и продавца. При `401/403` сессия считается
истёкшей; гостевая цена не подставляется. При временном `429/498` проверка завершается и
повторяется в следующем цикле без запуска Chromium. Штатный цикл не держит браузер постоянно
запущенным и не подменяет персональную цену публичной.

Цена зависит от браузерного региона/ПВЗ. При входе нужно выбрать тот же адрес, который
используется для покупок.

## Лицензированный fallback MPSTATS

При настроенном API token адаптер может запросить документированный MPSTATS endpoint
`/api/analytics/v1/wb/items/{id}/full`, когда публичный запрос WB недоступен. Проверяются
артикул, цена, остаток и свежесть `updated`. Это резерв только для общей цены артикула:
account-цена и выбранный размер на него не переключаются. Доступность полей и лимиты зависят
от действующей подписки MPSTATS.

## Известные ограничения

- DOM-селекторы и endpoint внутренние;
- точность персональной/WB Wallet цены зависит от полей внутреннего card API;
- финальная checkout-цена может отличаться;
- действующая оферта Wildberries прямо запрещает автоматический сбор цен и наличия; для
  эксплуатации нужен письменный допуск или лицензированный источник.
